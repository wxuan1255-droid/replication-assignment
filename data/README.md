# Data

## Source and access

This replication uses the two analytic Stata datasets from the original author's
public replication package, deposited alongside:

> Anderson, Michael L. (2017). "The Benefits of College Athletic Success: An
> Application of the Propensity Score Design." *The Review of Economics and
> Statistics*, 99(1), 119-134. https://doi.org/10.1162/REST_a_00589

The replication package is publicly available (Harvard Dataverse, linked from the
published article's supplemental materials) and **contains no confidential,
proprietary, or personally identifiable data** -- it is school-level and game-level
aggregate data (donation totals, admissions statistics, game scores and betting
lines), not individual-level records. Because of this, the two required files are
included directly in this repository under `data/raw/`, so the pipeline is fully
self-contained and reproducible without any manual download step.

## Files

| File | Description | Rows | Source (per original paper) |
|---|---|---|---|
| `raw/covers_data.dta` | Game-by-game football results and bookmaker point spreads, 1985-2010 | 31,656 | Covers.com |
| `raw/college data.dta` | School-year alumni donations, admissions, and academic-reputation data, 1986-2009 | 2,807 | VSE survey, IPEDS, US News & World Report |

If you obtain these files independently rather than from this repository, place
them in `data/raw/` with exactly these filenames (note the space in
`college data.dta`) before running `code/01_data_cleaning.py`.

## Generated files

`data/processed/analytic_panel.pkl` is generated by `code/01_data_cleaning.py` and
is not committed to version control (see `.gitignore`) -- it is fully reproducible
by running that script, and is regenerated fresh each time the pipeline runs.

## Key variable-coding notes (used by `01_data_cleaning.py`)

- `appdate` / `satdate` (reporting-date codes in `college data.dta`): `1` = value
  reported for the enrolled class of the *following* fall; `2` = value reported for
  the *current* fall's enrolled class; negative values are missing. See the
  `resolve_reporting_dates()` function for how these are resolved.
- `bcs`: indicator for membership in one of the six historically elite "Bowl
  Championship Series" conferences at the start of the sample period; recoded for
  four schools that changed conference affiliation mid-sample (see
  `recode_bcs()` and the paper's footnote 3).
- `team` / `teamname`: numeric and string school identifiers in `covers_data.dta`;
  `teamname` is the join key used to merge with `college data.dta`.

## Data-access limitations

None. Both files are public, non-restricted, aggregate data. This replication does
**not** rebuild these two files from the underlying raw IPEDS/VSE/US
News/Covers.com extracts (which are also referenced in the original author's
larger replication package but were not needed here); it starts from the already-
cleaned analytic datasets, which is documented as a scope limitation in the
replication report (Section 3, "Data" -- data-access limitations).
