## Result being replicated

**Primary result:** Table 3 of the original paper -- the estimated causal effect
of an additional football win on ten institutional outcomes (alumni athletic
donations, non-athletic donations, total donations, the alumni giving rate,
academic reputation, applications, the acceptance rate, out-of-state enrollment,
in-state enrollment, and 25th-percentile SAT scores), estimated two ways: a
Sequential Treatment Effects (STE) model and a propensity-score IV/matching
model.

**Extended beyond the minimum requirement:** this replication additionally
reproduces Table 1 (summary statistics), Tables 4-5 (the same Table 3 model
estimated separately for BCS and non-BCS schools), Table 6 (a robustness check
excluding early-season games), a version of Figure 1 (the binned relationship
between wins and outcomes), and the paper's concluding "three-win scenario"
economic-magnitude calculation.

Full details, including the exact estimating equations, are in
`report/replication_report.pdf`.

## How to run

**Requirements:** Python 3.10+ (developed and tested on 3.12). Install
dependencies with:

```bash
pip install -r requirements.txt
```

**Run order** (must be run in this order; each stage reads the previous
stage's output):

```bash
cd code
python 01_data_cleaning.py     # ~30 seconds -> writes ../data/processed/analytic_panel.pkl
python 02_analysis.py          # ~2 minutes  -> writes ../output/tables/*.csv (raw replicated results)
python 03_tables_figures.py    # ~15 seconds -> writes ../output/tables/comparison_all.csv
                                #                and ../output/figures/*.png
```

All three scripts use relative paths (`../data/...`, `../output/...`) and must
be run from inside the `code/` directory, exactly as shown above. No manual
path edits, environment variables, or credentials are required. No randomness
is used anywhere in the pipeline (every estimator is a deterministic maximum-
likelihood or least-squares computation), so no random seed is needed.

Alternatively, open `notebooks/replication.ipynb` and run all cells -- it calls
the same three scripts in sequence and displays the final comparison table and
figures inline.

## Obtaining and organizing the data

The two required source files are public, non-restricted, school- and game-
level aggregate data (no individual-level or personally identifiable
information), and are included directly in `data/raw/` so the pipeline runs
with no manual download step. See `data/README.md` for the full data
dictionary, provenance, and variable-coding notes.

## Expected output

After running all three scripts, `output/tables/` contains:
- `table1.csv` -- summary statistics
- `table3_all.csv`, `table4_bcs.csv`, `table5_nonbcs.csv`, `table6.csv` -- raw
  replicated STE and IV coefficients for each sample
- `comparison_all.csv` -- replicated vs. published coefficients, with sign-
  agreement and sample-size-match flags, for all 40 outcome x table estimates

and `output/figures/` contains:
- `fig_parity.png` -- replicated vs. published coefficients (scatter, both models)
- `fig_binscatter.png` -- binned relationship between wins and six outcomes

Headline result when this repository's code is run: the replicated STE
coefficients correlate with the published values at **r = 1.000** (40/40
matching signs), and the replicated IV coefficients correlate at **r = 0.998**
(38/40 matching signs, with both exceptions being published coefficients that
themselves round to 0.0000). Full detail in the report's "Replication Accuracy"
section.

## Known replication limitations

- **Table 2** (placebo/covariate-balance test) and **Appendix Table A1**
  (unexpected wins vs. losses) are not reproduced; both would reuse the
  estimator already implemented in `02_analysis.py`.
- **Table 7** (two-year persistence) is not reproduced -- it requires chaining
  three separate estimates together, a step the original author's own
  materials flag as requiring manual post-processing even in Stata.
- **Figure 1's exact confidence bands** (a kernel-weighted local polynomial
  regression with bootstrapped confidence intervals in the original) are
  approximated here with a binned scatter plot and a weighted linear fit,
  which recovers the same qualitative shape but not the exact original
  estimator.
- The **IV/matching model** is a from-scratch reimplementation of a custom
  Stata estimator rather than a byte-for-byte port, and its standard errors do
  not cluster by school at the bin level (unlike the original code); see the
  report's "Differences and Limitations" section for the full discussion,
  including a specific implementation detail (dependent-variable
  residualization on year fixed effects) that was not obvious from the paper's
  prose and had to be inferred from the original code.
- This replication starts from the two pre-cleaned analytic Stata datasets
  rather than rebuilding them from raw IPEDS/VSE/US News/Covers.com source
  files.

See `report/replication_report.pdf`, Section 7 ("Differences and Limitations"),
for full discussion.
